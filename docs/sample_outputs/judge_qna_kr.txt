[SafeGuard TwinLab Agent - 심사위원 예상 Q&A (자동 생성)]

Q1. 이 전력 값은 실제로 측정한 건가요?
A1. 아니요. datasheet 기반 추정 전력 모델(estimated power model)입니다. 실측이 필요하면 power_measure_log.csv를 업로드해 교차 검증할 수 있는 경로를 이미 구현해 두었습니다.

Q2. UWB/mmWave가 실제 센서 신호인가요?
A2. 아니요. raw RF/radar signal이 아닌 output-level virtual sensor simulation입니다. presence, distance, velocity 같은 최종 출력값만 시나리오 조건에 따라 합성합니다.

Q3. TI IWR6843ISK 같은 실제 하드웨어를 사용하신 건가요?
A3. 아니요. 이번 실행은 C4001 24GHz Low-cost profile(low-cost 24GHz radar module profile 기반 output-level virtual sensor simulation)입니다. 해당 모듈이 보고하는 output field 구성과 confirmation 임계값만 참고한 것이며, 실제 하드웨어나 raw signal processing을 사용한 것이 아닙니다.

Q4. 지금 보여주는 데이터가 dummy인가요, 실제인가요?
A4. 이번 실행은 dummy(합성) hardware log입니다. seed가 고정되어 있어 재현 가능하며, 실제 hardware_log.csv를 업로드하면 동일 파이프라인으로 즉시 재계산됩니다.

Q5. All-on 모델과 비교하는 게 SafeGuard에 유리하게 편향된 것 아닌가요?
A5. All-on은 SafeGuard와 동일한 센서/액추에이터 구성을 상시 가동하는 방식으로, 현장에서도 흔히 채택되는 '단순 상시 감시' 방식입니다. 공정한 상한(upper bound) baseline이며 이번 시나리오에서 총 에너지는 0.675 Wh입니다.

Q6. duty-cycle이 실제로 얼마나 절감되나요?
A6. 이번 시나리오 기준 SafeGuard(continuous)는 All-on 대비 26.5%, SafeGuard(duty-cycle)는 50.6% 절감됩니다. 두 결과를 항상 함께 제시하여 정책 변경에 따른 순수한 절감 효과를 투명하게 보여줍니다.

Q7. 이 결과가 재현 가능한가요?
A7. 예. src/config.py의 RANDOM_SEED로 모든 난수를 고정하며, 동일 시나리오 조건이면 항상 동일한 이벤트 로그와 전력 timeline이 생성됩니다. tests/ 이하 pytest로 상태 전이와 전력 계산을 자동 검증합니다.
